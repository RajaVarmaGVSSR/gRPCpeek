# Screenshot Placeholders

The root README references these placeholder images. Replace them with real screenshots before publishing broadly, or update the README links if you prefer `.png` or `.jpg` files.

Recommended captures:

1. `app-overview.svg`
   - Full app window with the sidebar, open request tab, request editor, and response panel visible.
2. `workspace-request-editor.svg`
   - Focus on service discovery, collections, request tabs, and the JSON request editor.
3. `response-inspector.svg`
   - Show a successful response with metadata, status, timing, and response size.
4. `environments-variables.svg`
   - Show workspace settings with environments, variables, auth, or TLS configuration.
5. `streaming-workflow.svg`
   - Show client or bidirectional streaming with multiple messages and live output.

Suggested screenshot sizes:

- Hero image: 1600 x 1000 or similar.
- Table images: 1200 x 800 or similar.
- Use light mode unless you want to show both themes.
- Avoid exposing private endpoints, tokens, certificates, or internal proto names.
